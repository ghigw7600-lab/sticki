## 🎯 앱 이름 선정 과정

### 후보군
| 이름 | 컨셉 | 결과 |
|------|------|------|
| 콜보드 (Callboard) | Call + Board | 초기 이름 |
| 콜리 (Colly) | Call + ly | 후보 |
| 핑 (Ping) | 신호음 | 후보 |
| 딩동 (DingDong) | 알림음 | 후보 |
| 포스티 (Posty) | Post + y | 해외 혼동 우려 |
| **Sticki** | Sticky note | ✅ **최종 선택** |

### 선택 이유
- Sticky note(포스트잇)와 직접 연결
- 글로벌하게 "붙이는 메모" 연상
- 짧고 발음 쉬움
- 귀엽지만 촌스럽지 않음

## 🎨 컬러 옵션

### 3가지 후보

**A. 민트 (선택됨)**
```
메인: #20C997
서브: #E6FCF5
다크: #1A1A2E
```
→ 상쾌하고 깔끔, 카톡/인스타/페북과 차별화

**B. 코랄**
```
메인: #FF6B6B
서브: #FFF5F5
다크: #2D2D2D
```
→ 따뜻하고 현대적

**C. 앰버**
```
메인: #F59F00
서브: #FFF9DB
다크: #212529
```
→ 포스트잇 느낌 강조, 단 카톡과 유사

## 📋 프리셋 설계 배경

### 타겟별 니즈

**영업용** (인바운드/아웃바운드)
- 여러 번 통화하며 진행
- 계약까지 단계적 관리
- 담당자 구분 필요

**사무직용** (내근직)
- "처리"라는 범용적 개념
- 거래처 문의 → 알아보고 → 완료
- 계약 개념 없음

**사장님용** (1인샵)
- 네일/미용/문신/두피문신 등
- 예약 중심 업무
- "3시에 손님 옴" 기록

**단순관리** (누구나)
- 할일 → 완료
- 가장 심플

## 🔧 스마트 파싱 로직

### 인식 순서
1. **연락처 먼저** (어디에 있든)
   - 휴대폰: 010/011/016/017/018/019
   - 일반전화: 02/031/032 등
2. **이름 패턴**
   - 2~4글자 한글 + 직함(대표/사장/담당 등)
3. **회사명 추론**
   - (주), ㈜, ~샵, ~병원 등
4. **나머지 → 메모**

### 문제점 해결
| 문제 | 해결 |
|------|------|
| 순서 섞여도 인식 | ✅ 패턴 기반 추출 |
| 일반전화 인식 | ✅ 02/031 등 지원 |
| 구분자 없어도 인식 | ✅ 공백/쉼표 유연 처리 |
